실제 UDP 서버와 소켓을 실행하고 datagram 기록을 남기기

이 페이지는 교육용 재현이 아니라 실제 Node UDP 서버 소켓과 실제 UDP 소켓을 실행한다. 브라우저는 제어 UI 역할만 하고, 실제 datagram 송수신 기록은 파일과 라이브 로그에 남는다.

서버

실제 UDP 바인드

새 UDP 서버 소켓을 실제로 바인드하고, 외부 프로그램이나 클라이언트가 보내는 datagram을 주소, 포트, 바이트, 패킷 수 단위로 기록한다.

클라이언트

실제 UDP 소켓

브라우저가 raw UDP를 직접 열 수 없으므로, 이 버튼은 서버 프로세스에 실제 UDP 클라이언트 소켓을 생성하고 바인드하도록 요청한다. UDP에는 handshake가 없으므로 바인드 후 바로 datagram을 보낼 수 있다.

활성 UDP 클라이언트가 아직 없다.

Datagram 제어

전송 / 소켓 닫기
